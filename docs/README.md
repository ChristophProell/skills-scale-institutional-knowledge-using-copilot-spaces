# OctoAcme Project Management Docs

## Overview

This README provides a summary of the project management processes and quick links to all relevant OctoAcme documentation. OctoAcme uses an iterative, customer-focused approach featuring clear roles, documented checklists, and regular review cadences.

### Project Management Process Summary

- **Customer-first:** Value and usability drive priorities
- **Iterative delivery:** Small, testable increments
- **Clear ownership:** PM and Product Lead roles defined
- **Evidence-based:** Impact tracked and measured
- **Safety:** Psychological safety and learning culture

#### Project Lifecycle

1. **Initiation:** Stakeholder alignment, problem statement, one-pager
2. **Planning:** Backlog creation, risk management, release mapping
3. **Execution & Tracking:** Standups, PR workflow, metrics, escalation path
4. **Release & Deployment:** Checklist-driven, testing, smoke tests, rollback plan
5. **Retrospective:** Captures learnings, turns actions into improvements
